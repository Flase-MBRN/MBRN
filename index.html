✦ Design-Entscheidungen

Local-First Architecture

Alle MBRN-Systeme folgen demselben Architekturprinzip: Berechnungen, State und Outputs entstehen ausschließlich im Browser des Nutzers. Kein Backend, keine Datenbank, keine Abhängigkeit von externen Servern — außer für Echtzeit-Marktdaten. Das ist keine Einschränkung. Es ist eine bewusste Entscheidung.

🔒
Zero Data Egress
Keine personenbezogenen Daten verlassen den Browser
⚡
Instant Execution
Kein Server-Roundtrip — alle Berechnungen in Echtzeit
∞
Persistent Access
Kein Account, kein Ablaufdatum, kein Paywall
◈
Portable State
URL-kodierter State für Cross-Device ohne Login
