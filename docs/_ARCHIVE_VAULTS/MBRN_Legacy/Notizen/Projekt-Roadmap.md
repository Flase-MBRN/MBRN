# Projekt-Roadmap

1. LM Studio richtig einrichten, Modelle laden und lernen wie das Programm funktioniert.

2. Numerologie-Rechner als Website fertigstellen  
   (v1.0 soll alle pythagoräischen Numerologie-Zahlenprofile berechnen können – anhand von Vorname, Nachname und Geburtsdatum, optional Geburtsname).

3. SmartFileBrain fertigstellen.

4. Erstes sinnvolles, verkaufbares und skalierbares Projekt starten (passend zu meinen Skills).

5. Prüfen, ob eine sinnvolle KI-Integration einen echten Mehrwert bringt.

6. Projekt verfeinern und restliche relevante Features einbauen.

7. Projekt vollständig fertigstellen.


# Projekt-Stop-Regel

1. Jedes Projekt bekommt zuerst ein klares v1.0 Ziel (MVP – Minimal funktionierende Version).

2. v1.0 darf nur die absolut notwendigen Funktionen enthalten, damit das Projekt funktioniert.

3. Sobald v1.0 funktioniert, wird das Projekt als "fertig" betrachtet.

4. Neue Ideen oder Features werden nicht sofort eingebaut, sondern zuerst in eine Ideenliste geschrieben.

5. Erst nach einer Pause oder nach Beginn eines neuen Projekts wird entschieden, ob ein v1.1 Update sinnvoll ist.

6. Ein Projekt darf maximal eine begrenzte Entwicklungszeit für v1.0 bekommen (z.B. wenige Tage bis maximal 1–2 Wochen).

7. Ziel ist es, viele fertige Projekte zu haben, statt wenige große Projekte, die nie abgeschlossen werden.
