# /pillars/oracle/fusion/

**Status:** ACTIVE

Diese Zone fuehrt die bereits normalisierten Oracle-Signale mit Snapshot- und Backtesting-Kontext zu einer UI-lesbaren Oracle-Sicht zusammen.

## Verantwortung

- Zieltag und Tagesnummer in die Consumption-Form bringen
- Recommendation, Confidence und Sentiment in eine gemeinsame Oracle-Sicht ueberfuehren
- Accuracy aus dem Backtesting in die UI-nahe Fusion einziehen

## Grenze

- keine Multi-Source-API-Fusion
- keine Rohdatenaggregation aus externen Bridges
- keine zweite JS-Fusionslogik ausserhalb des Snapshot-Flows
